<task type="auto">
  <name>Task 1: Create Sentry helper module with context and metrics functions</name>
  <files>lib/monitoring/sentry.ts</files>
  <action>
Create `lib/monitoring/sentry.ts` with the following exports:

**setSyncContext()** - Async function that:
- Imports `getPendingMutationCount` from `@/lib/offline/mutation-queue`
- Gets pending count via `getPendingMutationCount()`
- Calls `Sentry.setContext("sync", { isOnline: navigator.onLine, pendingMutationCount, lastSyncTime })`
- lastSyncTime should be passed as parameter or default to null

**setSentryUser(user: { id: string; email: string })** - Function that:
- Calls `Sentry.setUser({ id: user.id, email: user.email })`

**clearSentryUser()** - Function that:
- Calls `Sentry.setUser(null)`

**trackSyncMetrics(progress: SyncProgress)** - Function that:
- Import `SyncProgress` type from `@/lib/offline/sync-processor`
- Calculate successCount = progress.current - progress.errors.length - progress.conflicts.length
- Emit `Sentry.metrics.count("sync.success_count", successCount)`
- Emit `Sentry.metrics.count("sync.failure_count", progress.errors.length)`
- Emit `Sentry.metrics.count("sync.conflict_count", progress.conflicts.length)`

**updateSyncContextOnNetworkChange()** - Function that:
- Adds event listeners for 'online' and 'offline' window events
- Calls `setSyncContext()` when network status changes
- Returns cleanup function to remove listeners

Mark file with 'use client' directive since it uses browser APIs (navigator.onLine, window events).

Import Sentry as: `import * as Sentry from "@sentry/nextjs"`

1. Add import at top:
```typescript
import { trackSyncMetrics, setSyncContext } from '@/lib/monitoring/sentry';
```

2. In `processAllMutations()` function, after the final progress is determined (after the for loop, after setting `progress.lastSyncedAt`):
- Call `trackSyncMetrics(progress)` to emit Sentry metrics
- Call `setSyncContext(progress.lastSyncedAt)` to update context with latest sync time

The metrics should be emitted within the sync execution context so they're trace-connected. Add the calls right before returning the final progress object.

Do NOT wrap in try/catch - if Sentry fails, let it fail silently (Sentry SDK handles this). The sync operation should not fail because monitoring failed.
