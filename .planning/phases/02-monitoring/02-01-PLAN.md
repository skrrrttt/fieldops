  <action>
Modify `next.config.ts`:
- Import `withSentryConfig` from `@sentry/nextjs`
- Wrap the existing `nextConfig` export with `withSentryConfig(nextConfig, sentryOptions)`
- sentryOptions should include:
  - `org: process.env.SENTRY_ORG`
  - `project: process.env.SENTRY_PROJECT`
  - `authToken: process.env.SENTRY_AUTH_TOKEN`
  - `sourcemaps: { deleteSourcemapsAfterUpload: true }` (security: don't expose source maps)
  - `silent: !process.env.CI` (quiet local builds, verbose in CI)
  - `widenClientFileUpload: true` (better stack traces)

Create `app/global-error.tsx`:
- "use client" directive (error boundaries must be client components)
- Default export function receiving `{ error, reset }` props
- useEffect that calls `Sentry.captureException(error)` when error changes
- Render a simple error UI with "Something went wrong!" message and a "Try again" button that calls `reset()`
- Must return full HTML structure (html, body tags) since this is the root error boundary

Note: This error boundary catches errors in the root layout. Individual route error.tsx files can handle route-specific errors.
